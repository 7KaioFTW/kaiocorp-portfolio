# Accessibility Rules (WCAG 2.1 AA)

## Required
- All images: descriptive `alt` text (not "image" or empty)
- All interactive elements: keyboard accessible (Tab, Enter, Space)
- Links: never use `<div onClick>`, always use `<a>` or `<button>`
- Color contrast: text/background >= 4.5:1 for normal text, >= 3:1 for large text
- Form inputs: associated labels, error messages linked via aria-describedby
- Modal/dialogs: focus trap, auto-focus first field, trap Tab key

## Semantic HTML
- Use `<header>`, `<nav>`, `<main>`, `<footer>` landmarks
- Headings: h1 once per page, nested h2-h6 in logical order
- Lists: use `<ul>`, `<ol>`, `<li>` for list content
- Tables: use `<th>` for headers, `<caption>` for table title
- Forms: use `<fieldset>` + `<legend>` for grouped inputs

## ARIA
- Only use aria-* when semantic HTML isn't available
- `aria-label` for icon-only buttons
- `aria-current="page"` for active navigation link
- `aria-live="polite"` for dynamic status updates (e.g., count-up animations)
- Avoid `aria-hidden` on interactive elements

## Testing
- axe-core or similar automated accessibility audits
- Keyboard-only navigation test (no mouse)
- Screen reader test: Narrator (Windows), VoiceOver (Mac), NVDA (free)
